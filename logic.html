1. 全体構造

見積書の合計金額は、以下の各項目の合計に消費税を加算したものです。

合計 = (導入支援 + ライセンス + DM + DMデザイン + 反響対応月額 + 反響対応初期 + カスタム) × (1 + 税率)

各項目は「含める / スキップ」をトグルで切り替え可能です。

2. 導入支援費用（オンボーディング）

初回オンボーディング作業の費用。デフォルト値は月額固定費合計の 2.5 ヶ月分 として自動算出されます。

導入支援費用（推奨） = (ライセンス月額 + DM月額 + 反響対応月額) × 2.5

サイドバーで手動上書き可能。係数は管理者が variables.json の onboarding.multiplierMonths で変更できます。

3. WHEREライセンス費用

ライセンス合計 = 月額単価 × 期間（ヶ月）

プランを選択すると plans[].monthly が単価に反映されます。「特別プラン」チェックを有効にすると備考欄にその旨が記載されます。

ターゲット数を超過した場合の単価は plans[].overage（円/カウント）として備考に表示されます。

4. DM費用

DM月額 = DMプラン単価 × 月間通数 DM合計 = DM月額 × 契約期間（ヶ月） DMデザイン構成費 = 面数 × 50,000

DMプラン単価は dmPlans[].unit（円/通）。デザイン費単価は designFeePerSide。

5. 反響対応費用

反響対応月額合計 = 月額単価 × 契約期間（ヶ月） 反響対応 初期費用 = 設定額（一括）

月間上限件数を超えた場合の追加費用ロジックは別途運用ルールに従います（自動算出対象外）。

6. 小計・税・合計

小計 = 上記すべての合計 消費税 = floor(小計 × 0.10) 合計 = 小計 + 消費税

消費税は 切り捨て（floor） です。税率は tax.rate で変更できます。

8. 変数の修正・履歴・ロールバックについて

すべての変数は GitHub リポジトリの variables.json で管理されています。

  • 修正：管理者は GitHub 上で Pull Request を作成し、レビュー後にマージすると全ユーザーに反映されます。
  • バージョン管理：すべての変更は Git の commit 履歴として記録されます。
  • ロールバック：任意の commit に戻すには、対象 commit を git revert するか、variables.json の内容をその時点に戻す PR を作成してください。
  • バリデーション：.github/workflows/validate-variables.yml が PR 時点でスキーマ違反を検出します。

運用詳細は README.md を参照してください。
